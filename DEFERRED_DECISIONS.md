# SIMPLE RPG - Deferred Decisions

## Purpose

This file tracks important topics that are deliberately left open for later.
It is not a general backlog.

## Deferred Means

- important
- not forgotten
- not required to lock today

It does not mean:
- active rule
- approved implementation
- hidden future scope
- task queue
- feature wishlist

## Scope Rule

Keep only consciously open project topics here.

Do not use this file for:
- routine TODO items
- implementation chores
- bug lists
- speculative idea dumps

Speculative ideas belong in `EXPERIMENT_BANK.md`.

## Active Deferred Topics

### Stage 2+ Differentiation

- How later stages differ systemically from stage 1
- Revisit after the first playable stage loop works

### Event System Depth

- Event text depth
- Event branching complexity
- Event-combat mixing
- Revisit after map/reward loop feels stable

### Weapon Family Expansion

- Additional families beyond the earliest core set
- Revisit after first weapon identities are tested

### New Status Families

- Any status beyond the initial compact pool
- Revisit only if current pool proves insufficient

### Build Family Expansion

- Additional build identities beyond the first small set
- Revisit after actual play reveals missing strategic space

### Boss Phase Complexity

- Multi-phase or threshold-heavy bosses
- Revisit after baseline boss readability is validated

### Meta Progression

- Any between-run progression system
- Revisit only after the core run loop stands on its own

### Combat Save

- Saving during active combat
- Revisit after safe-state save is stable and cheap

### Advanced Tooling

- custom editors
- simulation dashboards
- advanced content tooling
- revisit when manual maintenance becomes expensive

## Not Deferred

These are already locked and should not be treated as open:
- engine baseline
- JSON content baseline
- single source of truth
- definition/runtime separation
- command-driven flow
- safe-state save baseline
- named RNG stream policy
- minimal combat action set
