# SIMPLE RPG - Deferred Decisions

## Purpose

This file tracks important topics that are deliberately left open for later.
It is not a general backlog.

## Deferred Means

- important
- not forgotten
- not required to lock today

It does not mean:
- active rule
- approved implementation
- hidden future scope
- task queue
- feature wishlist

## Scope Rule

Keep only consciously open project topics here.

Do not use this file for:
- routine TODO items
- implementation chores
- bug lists
- speculative idea dumps

Speculative ideas belong in `EXPERIMENT_BANK.md`.

### Advanced Tooling

## Not Deferred

These are already locked and should not be treated as open:
- `3`-stage run structure with the current successful and early-fail duration targets
- center-start bounded exploration graph as the run-spine map model
- partial fog plus adjacency traversal as the exploration information model
- revisit allowed without repeat-farm value on resolved nodes
- one stage-local `key` plus one locked boss gate per stage
- compact build engine scope:
  - base shared inventory: `5`
  - equipped belt bonus: `+2`
  - small linear synergies
  - no combo explosion
- engine baseline
- JSON content baseline
- single source of truth
- definition/runtime separation
- command-driven flow
- safe-state save baseline
- named RNG stream policy
- minimal combat action set
